# 세종 민원 안내 AI — 시민 민원 통합 응대 플랫폼 MVP

> **"근거를 확인할 수 있고, 불확실하면 안전하게 넘기는 전입신고 안내"**

2026 고려대 세종 산업체 실습 프로젝트 **1팀**의 MVP 저장소입니다.
가상 공고 「AI 기반 시민 민원 통합 응대 플랫폼 구축」(2026-세종-0001)에 대한 입찰 제안을 실제 동작하는 앱으로 구현합니다.

---

## 1. 프로젝트 소개

시민이 일상 언어로 민원을 질문하면 AI가 24시간 행정 정보에 **근거**하여 안내하고, 필요한 서류·절차·담당 부서까지 연결하는 챗봇 서비스입니다.

기능을 넓게 펼치는 대신, 가장 빈번한 민원인 **전입신고 1개 도메인**을 깊게 완성합니다(좁고 깊게 전략). 모든 답변에 출처를 표기하고, 확신이 없으면 추측하는 대신 담당 부서로 안전하게 연결합니다.

## 2. 해결하려는 문제

> 밤 10시, 세종으로 이사 온 직장인이 "전입신고 뭐 필요하지?"를 검색한다.
> 블로그마다 답이 다르고, 콜센터는 닫혀 있다. 어떤 정보가 맞는지 확인할 방법이 없다.

| 문제 | 내용 |
|---|---|
| 시민 접근성 | 평일 근무시간에만 응대 → 야간·주말 공백 |
| 정보 정확성 | 안내 기준 산재, 출처 불명 정보 혼재 |
| 행정 효율 | 단순·반복 문의에 콜센터 인력 소모 |

공공 AI 챗봇의 성패는 기능 수가 아니라 **"답을 믿을 수 있고, 틀릴 때 안전한가"** 에 달려 있습니다. 이 신뢰 문제를 설계의 중심에 둡니다.

## 3. 주요 기능 (MVP)

화면은 **홈 · 챗봇 · 정보 조회** 세 개 탭으로 구성되며, 대표 사용자 흐름 3종이 챗봇에서 end-to-end로 동작합니다.

| 시나리오 | 동작 | 요구사항 |
|---|---|---|
| ① 정상 질문 | 근거 기반 답변 + **출처 배지·원문 스니펫** 표기, 절차·서류 단계 카드 | SFR-001 · 002 · 003 |
| ② 모호한 질문 | 단정하지 않고 **후속 질문(선택지 버튼)** 으로 조건을 좁힌 뒤 답변 | SFR-005 |
| ③ 폴백 질문 | 근거 부족 시 추측 대신 **담당 부서 연결** — 행정동 선택 → 샘플 주민센터 매칭 + 대화 요약 카드 | SFR-006 · 004 |

공통 품질 장치:

- **지식베이스** — 전입신고 안내 문서 + 전입신고 관련 주민등록법 핵심 조문을 마크다운으로 정제·탑재 (DAR-001)
- **골든 QA 15문항 전수 통과** — 정상 8·모호 4·폴백 3문항을 3축(답변 정확성 / 출처 적합성 / 폴백 적절성)으로 채점 (QUR-002)
- **다양성 검증** — 골든 밖의 표현이 다른 질문에도 오폴백 없이 대응하는지 별도 하니스로 점검
- **환각 방지** — 출처 없는 단정 답변 0건 원칙, 불확실 시 폴백 (SER-003 · COR-001)
- **응답 대기 중 진행 표시** — 답변 준비 단계를 쉬운 말로 안내해 체감 대기시간을 줄임
- **개인정보 최소수집·로그 비식별화** (SER-001 · 002)
- **접근성** — 큰 글씨·고대비 토글, 키보드 포커스 표시·본문 건너뛰기, 움직임 최소화 설정 존중 (QUR-001)
- **모바일 우선 반응형** — 넓은 화면 대응 포함 (COR-002)

## 4. 기술 스택

| 계층 | 기술 |
|---|---|
| 개발 도구 | Kiro AI |
| Frontend | React |
| Backend | Python (FastAPI) |
| LLM | Upstage Solar (국산 LLM) — 답변 생성·의도 라우팅 |
| 검색 / RAG | 마크다운 지식베이스 + Upstage 임베딩 기반 의미 검색(인메모리 코사인) |

> API 키가 없어도 결정형(키워드) 경로로 동작하도록 이중화되어 있어, 데모·오프라인 채점 중에도 서비스가 멈추지 않습니다.

## 5. 실행 방법

백엔드와 프론트엔드를 **각각 다른 터미널**에서 실행합니다. 백엔드는 반드시 **프로젝트 루트**에서 실행하세요(루트에 `main.py`가 있습니다).

```bash
# 1) 백엔드 — 프로젝트 루트에서
pip install -r requirements.txt
python -m uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

```bash
# 2) 프론트엔드 — 다른 터미널에서
cd frontend
npm install
npm run dev          # http://localhost:5173
```

- **환경 변수(선택)**: 루트 `.env`에 `UPSTAGE_API_KEY`와 `UPSTAGE_MODEL=solar-pro2`를 넣으면 Solar LLM 기반 의미 검색·답변이 켜집니다. 키가 없으면 결정형 경로로 동작합니다.
- 프론트엔드는 `/api` 요청을 백엔드(`127.0.0.1:8000`)로 프록시하므로, 챗봇·주민센터 조회를 쓰려면 백엔드를 먼저 켜세요.

### 품질 검증

```bash
# 골든 QA 15문항 (키 없이 결정형 경로로 채점 → 문항×3축 매트릭스 + 평균 응답시간)
python qa/run_golden_qa.py

# 다양성·안전성 검증 (.env 키가 있으면 의미 경로로 측정: 재현율·안전율·clarify)
python qa/run_robustness_qa.py
```
