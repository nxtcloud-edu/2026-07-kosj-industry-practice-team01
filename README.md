# 세종 민원 안내 AI — 시민 민원 통합 응대 플랫폼 MVP

> **"근거를 확인할 수 있고, 불확실하면 안전하게 넘기는 전입신고 안내"**

2026 고려대 세종 산업체 실습 프로젝트 **1팀**의 MVP 저장소입니다.
가상 공고 「AI 기반 시민 민원 통합 응대 플랫폼 구축」(2026-세종-0001)에 대한 입찰 제안을 실제 동작하는 앱으로 구현합니다.

---

## 1. 프로젝트 소개

시민이 일상 언어로 민원을 질문하면 AI가 24시간 행정 정보에 **근거**하여 안내하고, 필요한 서류·절차·담당 부서까지 연결하는 챗봇 서비스입니다.

기능을 넓게 펼치는 대신, 가장 빈번한 민원인 **전입신고 1개 도메인**을 깊게 완성합니다(좁고 깊게 전략). 모든 답변에 출처를 표기하고, 확신이 없으면 추측하는 대신 담당 부서로 안전하게 연결합니다.

## 2. 해결하려는 문제

> 밤 10시, 세종으로 이사 온 직장인이 "전입신고 뭐 필요하지?"를 검색한다.
> 블로그마다 답이 다르고, 콜센터는 닫혀 있다. 어떤 정보가 맞는지 확인할 방법이 없다.

| 문제 | 내용 |
|---|---|
| 시민 접근성 | 평일 근무시간에만 응대 → 야간·주말 공백 |
| 정보 정확성 | 안내 기준 산재, 출처 불명 정보 혼재 |
| 행정 효율 | 단순·반복 문의에 콜센터 인력 소모 |

공공 AI 챗봇의 성패는 기능 수가 아니라 **"답을 믿을 수 있고, 틀릴 때 안전한가"** 에 달려 있습니다. 이 신뢰 문제를 설계의 중심에 둡니다.

## 3. 주요 기능 (MVP)

대표 사용자 흐름 3종이 end-to-end로 동작합니다.

| 시나리오 | 동작 | 요구사항 |
|---|---|---|
| ① 정상 질문 | 근거 기반 답변 + **출처 배지·원문 스니펫** 표기, 절차·서류 단계 카드 | SFR-001 · 002 · 003 |
| ② 모호한 질문 | 단정하지 않고 **후속 질문(선택지 버튼)** 으로 조건을 좁힌 뒤 답변 | SFR-005 |
| ③ 폴백 질문 | 근거 부족 시 추측 대신 **담당 부서 연결** — 행정동 선택 → 샘플 주민센터 매칭 + 대화 요약 카드 | SFR-006 · 004 |

공통 품질 장치:

- **골든 QA 15문항 검증** — 정상 8·모호 4·폴백 3문항을 3축(답변 정확성 / 출처 적합성 / 폴백 적절성)으로 채점 (QUR-002)
- **환각 방지** — 출처 없는 단정 답변 0건 원칙, 불확실 시 폴백 (SER-003 · COR-001)
- **개인정보 최소수집·로그 비식별화** (SER-001 · 002)
- **접근성** — 큰 글씨·고대비 토글 (QUR-001)
- **모바일 우선 반응형** (COR-002)

## 5. 실행 방법

> 🚧 3주차(7/20~7/24) 개발과 함께 업데이트됩니다.

```bash
# Backend
cd backend
pip install -r requirements.txt
# .env에 UPSTAGE_API_KEY 설정
uvicorn main:app --reload

# Frontend
cd frontend
npm install
npm run dev
```

### 골든 QA 채점

```bash
cd backend
python eval/run_golden_qa.py   # 15문항 일괄 실행 → 문항×3축 매트릭스 + 평균 응답시간 출력
```
